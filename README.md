# Brave's Maze Tour（勇者探索迷宮）

四層迷宮探索遊戲。每一層要收集完所有鑰匙，再走到終點（Goal）才能進入下一層。

## 遊戲模式

開始時可選擇三種模式：

| 模式 | 地圖來源 | 說明 |
|------|----------|------|
| **[1] Easy（教學）** | 固定迷宮（讀檔 maze_*.txt） | 熟悉操作與遊戲規則 |
| **[2] Normal** | Recursive Backtracker (DFS) 生成 | 走廊長而蜿蜒、岔路少 |
| **[3] Hard** | Randomized Kruskal's 生成 | 岔路多、死路短，最考驗探索 |

Normal / Hard 每次開始都會生成不同的地圖（保證 Goal 必定可達）。

## 主選單

```
Play    選難度（Easy / Normal / Hard）後開始
Mode    開關額外模組
Replay  觀看已存檔的遊玩紀錄
Quit    結束遊戲
```

選單一律用**方向鍵操作**：**↑/↓** 移動游標（`>` 指向目前選項），**Enter** 選擇。在 Mode 畫面用 **Enter / Space / ←/→** 切換該模組開關。按 **Esc** 回到上一層（數字鍵 1~4 仍可直接選）。遊戲進行中按 **E** 會回到主選單（不會直接關閉視窗），要結束程式請在主選單選 Quit。

選單採彩色外框與置中標題，目前選項以高亮色塊標示；每次移動游標時該列會閃一下（選取脈動動畫）。

## 音效

附帶音效（[Sound.cpp](src/Sound.cpp)）：每個事件對應一個 `.wav` 取樣音檔（選單移動/選擇/返回、開關模組、撿道具、喝藥水、打怪、遇到打不死的強敵、升級、過關、陣亡、傳送、限時逾時）。音檔放在 `assets/sfx/`，**非同步**播放、不會卡住畫面。回放播放時會自動靜音（避免重建過程連續觸發）。

- **Windows**：用 `PlaySound`（winmm，Makefile 自動加 `-lwinmm`）。
- **Linux**：呼叫系統的 `aplay`（alsa-utils）或 `paplay`（PulseAudio）在背景播放——不需連結音訊函式庫，只要環境裝了其中一個即可。一般桌面 Linux 都內建。
  - ⚠️ **WSL 注意**：WSL 預設沒有音訊裝置。Windows 11 的 WSLg 內建 PulseAudio 可直接出聲；Windows 10 的 WSL 需自行設定 PulseAudio 才會有聲音（沒設定時程式不會報錯，只是沒聲）。

音檔不是錄音，而是用 [tools/gen_sfx.py](tools/gen_sfx.py) 以正弦/三角波 + 諧波 + 包絡合成出來的（柔和、非方波）。想換音色就改腳本參數後重跑 `python tools/gen_sfx.py` 重新產生。

## 額外模組

在 Mode 畫面開關（兩者互斥）：

| 模組 | 說明 | 分數 |
|------|------|------|
| **Darkness Mode（全黑）** | 只看得到周圍 8 格，走過不留紀錄；HUD 顯示 `[Dark]` | ×1.5 |
| **Full-bright Mode（全亮）** | 整張地圖隨時可見；HUD 顯示 `[Bright]` | ×0.5 |
| **Monster Mode（怪物）** | 地圖上出現多隻怪物（每隻有等級，顯示在身上） | — |
| **Time Attack（限時）** | 每層有時間限制（`45 + 30×樓層` 秒），畫面顯示剩餘秒數，逾時即失敗 | ×1.5 |

Darkness 與 Full-bright 互斥；Monster、Time Attack 可與任一自由搭配，加成相乘（例：Darkness + Time = ×2.25）。

> 畫面採 **ANSI 顏色**（恆開，非模組）：牆灰、鑰匙/終點黃、傳送門青、怪物/障礙紅、經驗瓶綠、HP 藥水粉、勇者亮綠，HP 條紅、EXP 條綠。
>
> **動畫（恆開）**：畫面採無閃爍重畫（游標歸位 + 單幀輸出）。撞到怪物時該格紅底閃 3 下；撿鑰匙/喝藥水時該格反白閃 1~2 下；傳送門會持續脈動（`@` ↔ `*`）。傳送門脈動需要畫面定時刷新，所以遊戲主迴圈改為輪詢（約 180ms 一幀），按鍵反應不受影響（偵測到按鍵即時處理）。

### 怪物規則

- 怪物隨機散佈在地圖上，數量約為「樓層 + 1」，等級範圍 `1 ~ 2×樓層`（第 1 層 1-2、第 2 層 1-4、第 3 層 1-6、第 4 層 1-8）。等級依「玩家撿經驗瓶後各層可達的等級」調校，使大宗可擊敗、強敵需練級或繞過。
- **HP 戰鬥制**：怪物有血量（= 等級 × 20）。撞它一次：怪物扣掉你的 ATK；
  - 怪物**還沒死** → 反擊扣你 HP（= 等級 × 5），你停在原地（可選擇繼續打或撤退）。
  - 怪物**HP 歸零** → 擊敗，獲得 `40 + 等級×10` EXP（升級時 maxHP +20、ATK +5、回滿血），走上其位置。
- **等級壓制**：怪物等級比你高 **3 級（含）以上** 時打不死——你的攻擊對牠無效，每撞一次只會白白挨牠一發反擊，畫面會提示繞過。等你練級到差距 < 3 級就能開打。實戰落點：同級約掉 2~5 成血、+2 級約 7~8 成（非常吃力）、+3 級以上不可戰勝。
- HP 歸零 → 勇者陣亡，遊戲結束（顯示死亡畫面後回主選單）。
- ATK 越高（升級得來）越快解決怪物、挨打回合越少；強敵血厚反擊重，低等硬拚會被磨死。
- 怪物只會生成在**不擋住鑰匙/終點唯一通路**的位置，所以一定打得通關；強敵（含不可戰勝者）都可繞過。

### HP 藥水（HP Potion）

- 怪物模組開啟時，每層另外散佈「樓層」個 HP 藥水（外觀 `|H|`），撿到回復 **40 HP**（不超過上限）。
- 與經驗瓶一起構成戰鬥的補給：經驗瓶讓你變強、HP 藥水讓你撐住。

### 經驗瓶（EXP Potion）

- 怪物模組開啟時，每層另外散佈「樓層 + 1」個經驗瓶（外觀 `|E|`）。
- 每瓶固定增加 **100 EXP**。由於升級所需經驗隨等級變大（100、150、200…），同樣的 100 EXP 在低等級幾乎是一整級（Lv1 直接升級），高等級只填一小格 → **低等收益大、高等收益小，自然遞減**，高等級寧可去打怪。
- 是提供給玩家在低等時快速起步、追上怪物等級的途徑。

> 註：題目範例寫「怪物等級 ≥ 玩家等級就結束」，但那樣 Lv1 起手會無法靠戰鬥升級；此處採「> 才死、≤ 可擊敗」的可玩版本（規則寫在 `Block.cpp` 的 `Monster::player_touched`，要改回原版只需把 `>` 改成 `>=`）。

## 回放系統（Replay）

每次完整通關後，會把該局的**亂數種子、難度、模組、按鍵序列、時間與分數**，以時間命名（如 `replay_20260522_180449_111.txt`）存到本機 `replays/` 資料夾，一局一檔。在主選單選 Replay 會列出最近的紀錄（最新在上），選一局即可觀看——程式用同一個種子重建迷宮並依序重播當時的操作，完全重現原本的過程（HUD 顯示 `[REPLAY]`）。

回放時可操作：

| 按鍵 | 動作 |
|------|------|
| `Space` | 播放 / 暫停 |
| `D` | 前進一步 |
| `A` | 後退一步 |
| `W` / `S` | 加快 / 放慢播放速度（0.25x / 0.5x / 1x / 2x / 4x，以倍速顯示） |
| `Q` / `Esc` | 結束回放，回到 Replay 選單 |

`replays/` 為本機資料，已被 `.gitignore` 忽略。

## 分數計算

通關（四層全破）後會結算分數，公式：

```
分數 = (難度基底 + 時間獎勵 + 怪物獎勵 + 經驗獎勵) × 模組加成
```

| 項目 | 規則 |
|------|------|
| **難度基底** | Easy 1000 / Normal 2500 / Hard 4000（分數主體） |
| **時間獎勵** | 比該難度的目標時間（Easy 120s / Normal 210s / Hard 330s）每快 1 秒 +8 分，上限 1500；沒快過目標則為 0 |
| **怪物獎勵** | 每擊倒一隻怪物 +80 |
| **經驗獎勵** | 全程累積經驗值 ÷ 5（含打怪與經驗瓶） |
| **模組加成** | Darkness ×1.5、Full-bright ×0.5（無模組 ×1.0） |

怪物獎勵與經驗獎勵只在開啟 Monster 模組時才會累積（其餘模式沒有怪物/經驗瓶，這兩項為 0）。結算畫面會列出每一項與「擊殺數 / 總經驗」。

時間獎勵採「目標時間（par）」制：以難度基底為主，跑得比目標快才有額外加分，避免時間項蓋過難度。結算畫面會列出每一項明細；常數集中在 [Score.cpp](Score.cpp) 開頭，方便調整手感。

## 編譯與執行

即時鍵盤輸入採跨平台實作（Linux 用 `termios`，Windows 用 `conio.h` 的 `_getch`），兩邊都能編譯執行。

### Linux（demo 環境）

```bash
make            # 編譯 src/，產生 ./brave_tour
./brave_tour    # 在根目錄執行
make clean      # 清除編譯產物
```

### Windows

```powershell
g++ -std=c++11 -static -o brave_tour.exe src/*.cpp -lwinmm
.\brave_tour.exe
```

> `-lwinmm` 是音效（`PlaySound`）所需；用 `make` 編譯時 Makefile 會自動在 Windows 加上。

> Normal / Hard 模式為程式生成，不需地圖檔；Easy 模式才會讀 `maps/`。

## 操作方式

| 按鍵 | 動作 |
|------|------|
| `W` | 向上移動 |
| `A` | 向左移動 |
| `S` | 向下移動 |

## 玩家資訊（HUD）

畫面下方即時顯示：

- **Floor**：目前樓層 / 總樓層
- **Keys**：已收集 / 應收集鑰匙數
- **Time**：遊戲經過時間
- **Lv**：勇者等級（1~15）
- **ATK**：攻擊力（升級時提升）
- **HP**：血量條 + 數值
- **EXP**：經驗值條 + 數值（集滿升級、回滿血、攻擊力上升）

## 遊戲畫面元素

| 圖案 | 意義 |
|------|------|
| `o` `/|\` | 勇者（移動時外觀會改變） |
| `###` | 牆壁，無法通過 |
| `?` | 尚未探索的格子（fog of war） |
| `$` | 終點 Goal（第 3、4 層會隨機移動） |
| `o / -|-` | 鑰匙 Key，撿起後消失 |
| `X` | 障礙物 Obstacle，需用攻擊力打破 |
| `@ O @` | 傳送門 Portal，成對出現 |

## 遊戲規則

1. 一開始只看得到勇者周圍 8 格，移動後會逐步揭開地圖，探索過的格子不會再變回未知。
2. 第 N 層有 N 把鑰匙，隨機散落在地圖上（每次載入位置都不同）。
3. 撞到障礙物時，每次扣 10 點 HP（障礙物 HP 為 10~50），歸零後消失才可通過。
4. 踩到傳送門會被送到另一個傳送門的位置。
5. 收集完該層所有鑰匙後走到終點，即可進入下一層；四層全破即獲勝。

## 程式架構

```
Block（基底：3x3 symbol、是否可見、type、player_touched）
 ├─ Empty / Wall / Goal      （助教提供的基本元素）
 │    └─ MovableGoal          （會移動的終點）
 ├─ Key                       （鑰匙）
 ├─ Obstacle                  （障礙物）
 └─ Portal                    （傳送門）

Player：座標、鑰匙數、攻擊力 ATK=10、動畫、計時
Maze  ：vector2d<Block*> 地圖、讀檔、隨機放置道具、渲染畫面
```

碰撞行為以多型的 `player_touched()` 實作：`Player::move()` 呼叫目標方塊的
`player_touched()`，由各方塊自己決定結果（撞牆、撿鑰匙、打障礙、傳送等）。
